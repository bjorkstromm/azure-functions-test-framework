# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in this project, please report it through one of the following channels:

### Option 1 — GitHub Private Vulnerability Reporting (preferred)

Use GitHub's built-in private vulnerability reporting:

1. Go to the [Security Advisories](../../security/advisories) tab of this repository.
2. Click **"Report a vulnerability"**.
3. Fill in the details and submit.

Your report will be sent privately to the maintainers. We will acknowledge receipt within **5 business days** and aim to provide a fix or mitigation within **90 days**, depending on severity.

### Option 2 — NuGet Contact

If you prefer to report via NuGet, visit the [azure-functions-test-framework NuGet profile](https://www.nuget.org/profiles/azure-functions-test-framework), select the affected package, and use the **"Report package"** link provided by NuGet.

## Supported Versions

Security fixes are applied to the **latest released version** only. We encourage users to stay up to date.

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |
